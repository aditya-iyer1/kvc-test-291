# KV Cache Compression — Experimental Pipeline

End-to-end scripts for reproducing the KV cache compression analysis on LongBench using
**KVCache-Factory** with Llama-3-8B-Instruct.

## Requirements

| Requirement | Version                                     |
| ----------- | ------------------------------------------- |
| Python      | ≥ 3.10                                      |
| CUDA GPU    | ≥ 24 GB VRAM (A100-40G / A6000 recommended) |
| Disk space  | ~30 GB (LongBench data + model)             |
| PyTorch     | ≥ 2.1                                       |

## Directory Layout

```
291_proj/
├── scripts/
│   ├── 00_setup.sh             ← Install deps, download data
│   ├── 01_run_eval.sh          ← Run LongBench evaluation for all methods/budgets
│   ├── 02_score.sh             ← Score predictions (F1 / ROUGE-L / ExactMatch)
│   ├── 03_speedup.sh           ← Latency benchmark & speedup computation
│   ├── 04_plot.py              ← Generate all figures (5 PNGs)
│   ├── 05_generate_report.py   ← Compile final Markdown report from results
│   └── config.env              ← Auto-generated paths/settings (do not edit manually)
├── results/                    ← Created by the scripts
│   ├── budget_10pct/
│   ├── budget_20pct/
│   ├── budget_50pct/
│   ├── scores/summary.json
│   ├── timing/latency_report.json
│   └── figures/
└── kv_cache_compression_analysis.md  ← Final report (generated by script 05)
```

## Quick Start

> Run each step in order on your GPU machine from the `291_proj/` directory.

### Step 0 — Setup

```bash
cd /path/to/291_proj
bash scripts/00_setup.sh --model_path /path/to/Meta-Llama-3-8B-Instruct
```

This:

- Clones KVCache-Factory into `./KVCache-Factory/`
- Installs Python dependencies (including `flash-attn`)
- Downloads all 16 LongBench datasets from HuggingFace Hub
- Writes `scripts/config.env`

> **Tip:** If you're loading the model from HuggingFace Hub (not a local path), set
> `MODEL_PATH=meta-llama/Meta-Llama-3-8B-Instruct`. Make sure you've accepted the
> Llama-3 license and have `huggingface-cli login` configured.

---

### Step 1 — run evaluations

```bash
# Run everything (all 4 methods + FullKV baseline, 3 budgets, 16 datasets ≈ 3-8 hrs)
bash scripts/01_run_eval.sh --gpu 0

# Or narrow scope while testing:
bash scripts/01_run_eval.sh --gpu 0 --methods PyramidKV,SnapKV --budgets 0.2
```

Output: `results/budget_*/` — one `<METHOD>_<DATASET>.json` per combination.
Already-completed files are skipped automatically (safe to resume after interruption).

---

### Step 2 — Score predictions

```bash
bash scripts/02_score.sh
```

Applies the standard LongBench metrics (F1, ROUGE-L, ExactMatch).  
Output: `results/scores/summary.json` + per-budget JSON files.

---

### Step 3 — Measure speedup

```bash
bash scripts/03_speedup.sh --gpu 0 --seq_len 4096
```

Runs a latency benchmark over all method/budget combinations using a synthetic 4096-token
prompt (50 decode steps, 3 timed trials each).

Output: `results/timing/latency_report.json` and `results/timing/speedup_table.txt`.

---

### Step 4 — Generate figures

```bash
python3 scripts/04_plot.py
```

Produces 5 figures in `results/figures/`:

| File                           | Content                                                 |
| ------------------------------ | ------------------------------------------------------- |
| `fig1_accuracy_by_budget.png`  | Line chart: overall accuracy vs cache budget per method |
| `fig2_category_heatmap.png`    | Heatmap: method × task category at 20% budget           |
| `fig3_speedup.png`             | Bar chart: speedup at each budget                       |
| `fig4_accuracy_vs_speedup.png` | Pareto scatter: accuracy vs speedup                     |
| `fig5_dataset_radar.png`       | Radar chart: category profile at 20% budget             |

---

### Step 5 — Compile report

```bash
python3 scripts/05_generate_report.py
```

Reads `summary.json` and `latency_report.json`, fills in all result tables, and writes
`kv_cache_compression_analysis.md`.

---

## Methods Evaluated

| Method           | Paper                      | Key Idea                               |
| ---------------- | -------------------------- | -------------------------------------- |
| **FullKV**       | —                          | Full KV cache (baseline)               |
| **StreamingLLM** | Xiao et al., ICLR 2024     | Sink tokens + sliding window           |
| **H2O**          | Zhang et al., NeurIPS 2023 | Cumulative attention-score eviction    |
| **SnapKV**       | Li et al., NeurIPS 2024    | Observation-window clustering per head |
| **PyramidKV**    | Cai et al., EMNLP 2024     | Layer-adaptive budget (pyramid)        |

## Cache Budget Mapping

The `--max_capacity_prompts_ratio` flag controls the fraction of the effective context
length retained. At Llama-3-8B's effective context of 7,950 tokens:

| Budget | Ratio | Approx. Tokens |
| ------ | ----- | -------------- |
| 10%    | 0.10  | 795            |
| 20%    | 0.20  | 1,590          |
| 50%    | 0.50  | 3,975          |

## Notes

- **Attention backend:** Flash Attention 2 is used by default (set in `config.env`).
  For V100 or RTX 3090 (no FA2), change `ATTN_IMPL=sdpa` in `config.env` before
  running steps 1–3.
- **H2O compatibility:** H2O is implemented with SDPA in KVCache-Factory to avoid
  the FlashAttention materialization constraint, adding modest overhead.
- **Reproducibility:** Seed is fixed at 42 in all scripts. Results may vary slightly
  across GPU hardware due to floating-point nondeterminism.
